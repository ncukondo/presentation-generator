# スライドデザイン原則

このプレゼン生成器で出力するスライドに共通して適用するデザインルール。
`lib/render.ts` でレイアウト（layout）を書く時、QA エージェントが画像をチェックする時、この文書が判定基準になる。

原則は「優秀なデザイナが守る普遍則」ではなく「**このリポジトリで**いま守るべき運用ルール」として書く。曖昧な標語より、機械的に違反を検出できる具体値を優先する。

---

## 1. 配色の運用

### 役割を固定する — 色に「意味」を持たせる

1つの色 = 1つの役割。順番や装飾で色を割り振らない。
具体色は**テーマ（`slides.yaml` の `theme:` / `lib/themes.ts`）で差し替わる**ので、コードでは hex 直書きせず役割名（`C.xxx`）で参照する。既定 preset は `slate`。下表の hex は分かりやすさのため preset `blue` を例に示す（役割名で参照していれば preset 差し替えで一括再配色される）。

| 役割 | 参照（例: blue preset） | 使う場所 |
|------|--------|---------|
| **Primary** | `C.primary` (例 #1E88E5) | タイトルバー・章見出し・ボディ強調の主役 |
| **Accent** | `C.accent` (例 #FF9800) | CTA・重要告知・アンダーライン。**1スライドに最大1箇所** |
| **Text** | `C.text` (例 #37474F) | 本文。真黒は避ける |
| **MidGray** | `C.midGray` / `C.darkGray` | 補助情報・引用文字 |
| **Warm BG** | `C.warmBg` (例 #FFF9F2) | スライド地色 |
| **Card pastel** | `C.cardBlue` etc. | セマンティックに色分ける時**だけ**使う |
| **Step colors** | `C.step1..7` | 同一画面に並ぶ「区別が意味を持つ」要素（7ステップ・複数カテゴリ）専用 |

### 配色の禁止事項

- ❌ カードや番号を**順番で色分けしない**。内容が「並列3項目」なら全部同色（primary 濃淡・モノクロ）で揃える。
- ❌ 1枚のスライドで**6色以上**の顕色（primary/accent/step カラー）を使わない。pastel 背景は色数にカウントしない。
- ❌ 真っ赤 (#E53935) や真っ緑 (#43A047) のベタ塗りをカードヘッダに使わない。**ヘッダは primary または step カラーの濃いもの**に統一し、意味的に「警告・成功」の文脈でしか彩度の高い赤緑を出さない。
- ❌ 青背景の上に `C.accent` の小さい文字を置かない（コントラスト 2.5:1 以下）。

### 優越比 (dominance)

1スライドの面積比: primary 60–70% / 中立色（地色+text）20–30% / accent 5% 以下。
「全部色鮮やか」は散漫に見える。

## 2. 最小フォント・タイポグラフィ

### 最小サイズ

| 種別 | 最小 pt | 備考 |
|------|---------|------|
| 本文・カード内文 | **22pt** (`FS.body`) | これを下回らない |
| 補足・引用・注記 | **18pt** (`FS.micro`) | 画面の 5% 以下の面積に限る |
| タイトル | 28pt (`FS.slideTitle`) 以上 | |
| セクション見出し | 24pt (`FS.heading`) 以上 | |

> **ルール**: 本文で 20pt 以下を使いたくなったら、まず「テキスト量を削減」を試す。削減できない時だけ例外扱い。

### フォント

日本語は `Meiryo`、英語は `Arial`。1スライド内でフォントを3つ以上混在させない。

### 折り返し禁止ゾーン

- カード幅が「**1行に最低でも全角15文字**」入らない場合、そのレイアウトは採用不可。列数を減らすか、テキストを削る。
- 縦書きのような「1文字ずつ改行される」折返しが出たら**即修正**。
- **孤立行（orphan）**: 最終行が全角1〜3文字しか無い折返し（例「…タイト」/「ル」）は不可。
- 英数字トークン（`GPT-4o` 等）を途中で割らない。句読点 `。、）」` を行頭に出さない（禁則処理）。
- タイトルは2行以内。3行超は内容を削る。

> ビルド時に `lib/text-metrics.ts` が禁則・自然な区切りでテキストを自動バランスし、orphan を解消する。`\n` 挿入で直せないもの（幅不足・縦はみ出し・分割不能な長語）は `[wrap] RECOMMEND` と `wrap-report.json` に改修推奨として出る。

## 3. 余白・レイアウト

### 絶対マージン

- スライド端からコンテンツまで **0.5in 以上** (`MARGIN.left/right` ≥ 0.6in 遵守)
- カード間の隙間 **0.2in 以上**
- 要素同士の最小ギャップ **0.1in**（これ未満は「重なり」と判定）

### 垂直バランス

- 「上半分だけ使って下が空白」は禁止。**コンテンツ領域の上下70%以上**に要素を分布させる。
- 下部にスペースが余るなら、フォントを上げる・カードを縦に伸ばす・セクションを増やす・レイアウトを変える。

### 中央偏り禁止

- 全要素が画面中央に寄って左右に大きな余白、というレイアウトは NG。**コンテンツ領域の 80% 以上の幅**を使う。
- 例外: タイトルスライド・全画面引用（hero-quote）のみ。

---

## 4. レイアウトのバリエーション

**同じレイアウトを3枚以上連続で使わない。** ステップ詳細のように性質上同一になる場合を除き、以下から意識的に切り替える。

| レイアウト | 用途 |
|-----------|------|
| `title-hero` | タイトル・章扉 |
| `three-cards` | 並列3項目（背景の課題など） |
| `two-column` | 対比・定義+事例 |
| `icon-row` | 横並びのステップ/項目 |
| `half-bleed-image` | スクリーンショット・図を大きく見せる |
| `hero-quote` | 1枚まるごと大きな一文（章の締め） |
| `data-grid` | 2x2 / 3x2 の均質な項目群 |
| `timeline` | 時系列・プロセス |
| `references` | 文献一覧 |

各スライドは**画像・アイコン・図形のいずれか1つ以上**を含めること（純テキストだけのスライドは作らない）。

---

## 5. アイコン・図の扱い

- アイコン単体の最小サイズ **0.6in**。カード内の小さいフットプリントに置く時でも **0.4in** を下回らない。
- アイコンは「内容の比喩」であるべき。**装飾だけのアイコンを置かない**（slide02 の脳・時計・ヘッドフォン問題）。内容に合致するアイコンが無いなら置かない方がマシ。
- ステップ番号の丸アイコン (`addStepNumber`) は最小 **0.5in**。7ステップ並べる場合は 0.4in まで許容。

---

## 6. コントラスト要件

WCAG AA 基準を目安にする。

- 本文 vs 背景: **4.5:1 以上**
- 大きな文字（24pt 以上）・アイコン vs 背景: **3.0:1 以上**
- **青系の濃い背景に白以外の色で文字を置かない**。特に `accent` の橙は青背景でコントラスト不足（slide01 発表者名・slide14 補足メッセージの事例）。

---

## 7. 一貫性のビジュアル要素 (repeat element)

プレゼン全体で**1つの署名的なビジュアル要素**を繰り返し、統一感を出す。このリポジトリでは:

- **上部に primary 青の帯 + 下に accent 橙のアンダーライン** を全コンテンツスライドに適用
- カードは **角丸 0.08in の rectRadius** で統一
- 番号は **円形塗りの step カラー** で統一

新レイアウトを追加する時もこの要素を流用する。

---

## 8. コンテンツの衛生 (content hygiene)

- プレースホルダ文字列（`xxxx`, `TODO`, `lorem ipsum`, `ここに記入`, `〜をここに記載`）が**出力画像に残っていたら失格**。QA チェックで検出する。
- `cite()` が空文字列や未定義を受けた場合、その文字列自体を出さない（`(@)` 表示禁止）。
- References スライドが空になる状態は「引用未使用」または「引用キー未解決」のサインなので、warning を出す。

---

## 9. 違反時の判断フロー

QA で違反が見つかった場合、以下の順で検討する:

1. **コンテンツ削減**: テキストを短く・項目を減らす。**まずこれを試す**。
2. **レイアウト変更**: 別レイアウトに差し替える（3カラム→2カラム、grid→row）。
3. **フォント/余白調整**: 最後の手段。最小値を割る調整はしない。

サイズを無理に小さくして収めるより、**内容を減らす方が常に正解**。
